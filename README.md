# Multispectral Representation of DAS data
This project provides scripts to test the Multispectral Representation of DAS data (MsR-DAS) methods described in:

Morell-Monzó, S., Diego-Tortosa, D., Pérez-Arjona, I., Espinosa, V. (2026) **Multispectral Representation of Distributed Acoustic Sensing data: an approach for physically interpretable feature extraction and visualization**. ArXiv: https://doi.org/10.48550/arXiv.2604.07290

If you use this repository in your research, a citation to the manuscript would be appreciated.

## Requirements
After cloning the repository:
```
git clone https://github.com/sermomon/MsR-DAS.git
cd MsR-DAS
```

Python >= 3.11 is required. It is recommended to use a virtual environment:
```
# Create virtual environment:
python -m venv venv

# Activate environment (Linux / macOS):
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

Upgrade pip and install dependencies:
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## Main notebook

An example workflow is implemented in `MSR-DAS_example.ipyn`. 
This notebook serves as the main entry point of the project and some functionalities in a reproducible step-by-step manner.

### Pipeline description

#### 1. Environment setup, download example data and metadata reading:
The notebook first imports external dependencies and internal modules from `src/`.

Then, an example HDF5 file from the [OOI public dataset ](https://doi.org/10.58046/5J60-FJ89) is downloaded using the function `src/download_url_file.py`. The file is saved in the `results` directory and its metadata are printed for inspection.
 
This file contains a 60 s experimental recording featuring two Fin Whale vocalizations: type-A (25--19 Hz) and type-B (16--20 Hz).

#### 2. Data reading, crop data array and Spectrogram representation:
The downloaded HDF5 file is loaded using the DAS data reader from `src/`, selecting the desired spatial channels along the fiber. The loaded data array is characterized in terms of cable length, recording duration, spatial sampling interval (`dx`), and temporal sampling interval (`dt`).

The DAS data are then cropped to the region of interest, selecting a specific time window (0--40 s) and fiber section (10--60 km) for further analysis.

Also, a spectrogram showing the two different Fin Whale vocalizations is generated and saved in the `results` directory. The spectrogram computation uses the [LT-Acoustic-Feature-Extractor](https://github.com/ddietor/LT-Acoustic-Feature-Extractor "GitHub repository") tools, following the methodology described in [Effective Strategies for Automatic Analysis of Acoustic Signals in Long-Term Monitoring](https://doi.org/10.3390/jmse13030454 "DOI: 10.3390/jmse13030454")*.

#### 3. **MsR-DAS** application:
The Multispectral Representation (MsR) is generated by defining specific spectral bands and applying spectral decomposition.

Three frequency bands are selected to separate the different acoustic components:
- Red band (B1): Fin Whale type-B vocalizations (16--20 Hz)
- Green band (B2): Fin Whale type-A vocalizations (20--28 Hz)
- Blue band (B3): Low-frequency noise (3--15 Hz)

The `spectral_decomposition` function is applied to extract the contribution of each frequency band, and each band is normalized before RGB composition.

Finally, the conventional DAS representation (15--27 Hz) is compared with the proposed MsR representation. In the MsR image, the different Fin Whale vocalization types are highlighted by different colors, allowing type-B and type-A vocalizations to be visually distinguished in red and green, respectively.
